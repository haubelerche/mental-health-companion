## 3. Yêu cầu GitHub

**AI20K Build Phase**

### Yêu cầu GitHub

**Các hạng mục bắt buộc trong Repository**

### 1. Source Code

Repository cần có source code đầy đủ, bao gồm:

- Frontend.
- Backend.
- Database.
- Agent / AI logic.
- API.
- File cấu hình.
- Tài nguyên chạy dự án.

### 2. README.md

README.md cần:

- Đặt tại thư mục gốc.
- Có mô tả dự án.
- Có mục tiêu.
- Có tính năng.
- Có công nghệ sử dụng.
- Có hướng dẫn cài đặt.
- Có hướng dẫn chạy.
- Có hướng dẫn sử dụng.

### 3. Architecture

Repository cần có sơ đồ kiến trúc thể hiện:

- User.
- Frontend.
- Backend/API.
- Database.
- AI Agent/LLM.
- Luồng dữ liệu chính.

### 4. AI Logs

Repository cần có:

- Prompt mẫu.
- Chat logs.
- Cấu hình Webhook theo hướng dẫn của BTC.

### Thông điệp chính

**Repository rõ ràng giúp chấm kỹ thuật nhanh hơn.**

## 4. README & kiến trúc

**AI20K Build Phase**

### README & kiến trúc

**Giúp người chấm hiểu và chạy được dự án.**

### README.md cần có

- Tên dự án.
- Mô tả ngắn gọn.
- Mục tiêu / vấn đề giải quyết.
- Tính năng chính.
- Công nghệ sử dụng.
- Hướng dẫn cài đặt.
- Hướng dẫn chạy dự án.
- Hướng dẫn sử dụng sản phẩm.

### Architecture

Sơ đồ kiến trúc cần mô tả các thành phần chính và luồng dữ liệu:

```text
User -> Frontend -> Backend/API -> Database -> AI Agent/LLM -> External Services
```

Luồng dữ liệu chính cần được thể hiện rõ giữa các thành phần.

### Lưu ý quan trọng

Đặt các link quan trọng ngay phần đầu README.

### Thông điệp chính

**README rõ ràng = ít mất thời gian kiểm tra.**

## 5. Nhật ký & minh chứng

**AI20K Build Phase**

### Nhật ký & minh chứng

**Đừng chỉ nộp code, hãy nộp cả quá trình.**

### 1. Weekly Journal

Cần ghi lại theo từng tuần:

- Mục tiêu.
- Kết quả.
- Khó khăn.
- Cách team giải quyết.

### 2. Worklog

Cần có thông tin về:

- Thành viên phụ trách.
- Công việc.
- Thời gian.
- Trạng thái hoàn thành.
- Ghi chú.

### 3. Evaluation Evidence

Cần chuẩn bị minh chứng đánh giá, bao gồm:

- Báo cáo đánh giá.
- Kết quả test.
- Bộ câu hỏi kiểm thử.
- Metrics.
- Feedback.
- Ảnh chụp màn hình.

### Cần trả lời được

- Sản phẩm có đúng mục tiêu không?
- Agent có xử lý chính xác không?
- Hệ thống có ổn định không?
- Đã kiểm thử nhiều tình huống chưa?

### Thông điệp chính

**Minh chứng rõ ràng giúp bảo vệ chất lượng dự án.**

## 6. Nộp qua Form

**AI20K Build Phase**

### Nộp qua Form

**Chuẩn bị sẵn link trước khi Form mở.**

### 1. Live URL

Sản phẩm cần chạy ổn định trên môi trường deploy.

Ví dụ định dạng URL:

```text
https://a20-app-xxx.domain
```

Trong đó:

- `xxx` là tên/mã team viết thường.
- Không dùng dấu.
- Không dùng khoảng trắng.
- Domain có thể là bất kỳ domain deploy hợp lệ.

### 2. Video Demo

Video demo cần dài **3-5 phút**, thuyết minh:

- Bài toán.
- Tính năng chính.
- Luồng xử lý của AI Agent.

### 3. Pitch Deck

Pitch deck cần dài **5-10 trang**, gồm:

- Vấn đề.
- Giải pháp.
- Công nghệ.
- Kết quả.

### Lưu ý truy cập

Tất cả link Drive, YouTube, Slide cần mở quyền truy cập công khai.

### Thông điệp chính

**Kiểm tra quyền truy cập trước khi gửi Form.**
